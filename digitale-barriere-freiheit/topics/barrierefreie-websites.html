Barrierefreies Webdesign: Grundlagen und effektive Ansätze

Barrierefreie Websites sind ein wichtiger Bestandteil der modernen Welt. Sie ermöglichen es Menschen mit unterschiedlichsten Fähigkeiten auf Informationen aus dem Internet zuzugreifen. Das sorgt für Inklusion und Gleichberechtigung aller Menschen. Es ist in einigen Ländern sogar gesetzlich vorgeschrieben.

Farben und Kontraste

Auf barrierefreien Websites ist es wichtig starke Farben und Kontraste zu nutzen. So können sie auch Menschen mit Sehproblemen gut lesen. Es ist wichtig, dass die Farben klar sind und sich gut vom Hintergrund abheben.

Kontraste nach WCAG

Die WCAG ist eine Sammlung von Regeln für leicht benutzbare Webseiten. Eine dieser Regeln besagt, dass Texte einen ausreichenden Kontrast haben müssen. Sie empfehlen einen Kontrast von mindestens 4,5:1 für normalen Text und 3:1 für größeren Text. Diese Werte sagen aus wie stark sich die Farbe des Textes vom Hintergrund abhebt. Für einen sehr guten Kontrast sollte der Text einen Kontrast von 7:1 für normalen Text und 4,5:1 für größeren Text haben.

Wie setzt man Farben richtig ein?

Farben sind auch wichtig, weil sie unsere Gefühle beeinflussen können. Verschiedene Farben haben verschiedene Bedeutungen. Zum Beispiel steht Rot für Energie und Blau für Vertrauen. Aber es ist wichtig zu wissen, dass manche Menschen Farben anders sehen. Deshalb sollte man Informationen nicht nur durch Farben zeigen. Am folgenden Beispiel zeigt sich, dass allein durch Farbe keine Information barrierefrei aufnehmbar ist. Sobald die Farbe von den Balken verschwindet kann man die Balken nicht mehr zuordnen. Um für Barrierefreiheit zu sorgen muss die Information unabhängig von der Farbe aufgenommen werden können. Dieser Anspruch ist nicht umsonst auch ein WCAG-Kriterium.

Werkzeuge zum Überprüfen von Farbkontrasten

Es gibt spezielle Werkzeuge für das Überprüfen der Farbkontraste auf Webseiten. Diese Werkzeuge sind nützlich um sicherzustellen, dass eine Webseite für alle gut lesbar ist. Beim Gestalten einer Webseite sollte man immer an die Kontraste denken: Bei Texten, Logos, Buttons und anderen wichtigen Elementen.

Semantisches HTML: Das Gerüst jeder Website

HTML ist wie das Gerüst einer Webseite. Es sagt dem Computer, wo Texte, Bilder und andere Sachen auf der Seite sein sollen. Semantisches HTML ist eine besondere Art von HTML. Man benennt jeden Teil des Gerüsts, damit man genau weiß was es ist. Zum Beispiel sagt semantisches HTML dem Computer, dass ein bestimmter Teil der Seite die Überschrift ist oder dass ein anderer Teil das Hauptmenü ist. Das hilft vor allem Menschen, die spezielle Programme zum Lesen der Webseite benutzen.

Beispiele für semantische HTML Elemente

  • <header> und <footer>: Kopf- und Fußzeile der Seite.
  • <nav>: Navigationsbereich, oft mit Menü-Links.
  • <main>: Der Hauptinhalt der Seite.
  • <article> und <section>: Teile der Seite, die für sich stehen können.
  • <aside>: Informationen, die nicht direkt zum Hauptinhalt gehören.
  • <h1> bis <h6>: Überschriften, die die Struktur der Seite zeigen.

ARIA: Ein wichtiger Bestandteil für barrierefreiere Internetnutzung

ARIA (“Accessible Rich Internet Applications”) ist eine Technologie um Webseiten besser nutzbar zu machen. Besonders für Menschen, die spezielle Hilfsmittel wie Screenreader benutzen. Zum Beispiel werden durch AIRA Webseiten für Menschen mit Sehbehinderungen zugänglicher. Es besteht aus speziellen Codes, die man in die Webseite einbaut. Diese Codes sagen Hilfsmitteln wie Screenreadern, was auf der Seite passiert und wie man sie benutzen kann.

Warum AIRA so wichtig ist

Stellen Sie sich vor, Sie hören eine Webseite anstatt sie zu sehen. Ohne ARIA könnte es schwer sein zu verstehen, wo auf der Seite bestimmte Informationen stehen oder was ein Button macht. ARIA sagt dem Screenreader zum Beispiel, ob ein Button etwas einschaltet oder ausschaltet, oder wo der Hauptteil der Seite beginnt und endet.

Beispiele für AIRA

  • aria-label: Gibt einem Element einen Namen. Das ist nützlich, wenn der Text nicht klar sagt, was das Element macht. Zum Beispiel könnte ein Button nur ein Symbol haben. Mit aria-label können Sie erklären, dass der Button zum Schließen des Fensters dient.
  • role: Sagt, welche Rolle ein Element auf der Seite hat. Zum Beispiel kann ein Element die Rolle eines Buttons oder einer Navigation haben.
  • aria-live: Wird verwendet, um Bereiche zu kennzeichnen, die sich automatisch aktualisieren. So weiß der Screenreader, dass er diese Teile der Seite besonders beachten muss.

Der Dialog wird mit einem Button geöffnet. Er enthält einen Titel und einen Schließ-Button. Dank ARIA weiß eine Person, die einen Screenreader benutzt, dass sie sich in einem Dialog befindet. Das Attribut aria-modal="true" teilt dem Screenreader mit, dass der Nutzer auf den Dialog konzentriert bleiben sollte, bis er geschlossen wird.
